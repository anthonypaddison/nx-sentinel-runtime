# 2026-03-03

## What changed
- Made `NX - Family Dashboard` the only active Lovelace dashboard:
  - removed `nx-displaygrid` and `nx-displaygrid-v2` dashboard registrations from `config/configuration.yaml`,
  - removed legacy dashboard YAML files `config/lovelace/nx-displaygrid.yaml` and `config/lovelace/nx-displaygrid-v2.yaml`.
- Switched family-mode navigation paths to feature-flag-driven behavior:
  - `_allowedViews()` now derives optional family screens from enabled `v2_features`,
  - family sidebar/topbar menus now use enabled extra screens instead of hardcoded food/family/ambient entries,
  - ambient dock menu now builds its nav entries from the same enabled-screen source.
- Removed external screensaver image dependency:
  - added `config/www/nx-displaygrid/assets/screensaver-mountain.jpg`,
  - updated card screensaver CSS to load `/local/nx-displaygrid/assets/screensaver-mountain.jpg`.
- Updated runbooks/docs/scripts for the current dashboard baseline:
  - README + architecture + concept docs updated to reflect family-dashboard-first runtime,
  - focused export now includes `config/lovelace/nx-family-dashboard.yaml`,
  - restore/smoke-test docs updated to reference `NX - Family Dashboard`.
- Added a regression test for family-mode feature-flag view gating and bumped Lovelace resource cache-bust query string.

## Why it changed
- Current implementation and intended direction are family-dashboard-first; removing legacy dashboards and aligning docs/navigation behavior reduces UX ambiguity and keeps enabled features consistent with configured flags.

## How to verify
1. Run checks:
   - `bash scripts/check.sh`
   - `bash scripts/check_structure.sh`
2. Confirm only `NX - Family Dashboard` is registered:
   - open `config/configuration.yaml` and verify only `nx-family-dashboard` exists under `lovelace.dashboards`.
3. Confirm legacy dashboard YAML files are absent:
   - `config/lovelace/nx-displaygrid.yaml`
   - `config/lovelace/nx-displaygrid-v2.yaml`
4. In Home Assistant, hard refresh Lovelace and confirm only `NX - Family Dashboard` appears in sidebar.
5. In Settings, toggle `v2_features` values (for example `food_view`, `family_dashboard`, `ambient_view`) and verify family nav menus reflect enabled screens only.
6. Trigger screen saver and confirm the image loads from `/local/nx-displaygrid/assets/screensaver-mountain.jpg` with no external image fetch.
