# 2026-02-25

## Family dashboard follow-up polish

### What changed
- Family-mode nav now keeps `Admin`/`Audit` inside Settings tabs only:
  - Removed standalone `Admin`/`Audit` entries from family sidebar and topbar menus.
  - Added/kept Settings panel tabs for `Settings`, `Admin`, `Audit` with a shared `Save changes` button.
- Ambient follow-up UX:
  - Bin icon in upcoming row no longer uses a circle wrapper.
  - Fixed duplicate `Bin day` text rendering.
  - Moved `Heating`, `Lighting`, `Screensaver` to a bottom three-button equal-width action row.
  - Kept stat-tile routing to target screens (`Schedule`, `Chores`, `Shopping`, `Home`).
- Food follow-up UX:
  - Removed `In the house` tab.
  - Recipes now use `+` add flows for ingredients and steps (instead of free-text-only textareas).
  - Ingredient quantity defaults to `1` with default unit `quantity`.
  - Added meal-plan `Begin cooking` flow that opens a temporary `Cooking` tab for step checkboxes and hides again on `Finish cooking`.
- Settings follow-up safety:
  - Added two-step confirmation for shopping favourites reset/clear.
  - Second confirmation step intentionally swaps button placement to reduce accidental double-taps.
- Calendar and month view polish:
  - Add-event now shows `Event saved` toast on successful create.
  - Month-view today-events panel forced to a flexed vertical scroller to keep it scrollable.
- Updated Lovelace cache-bust resource to:
  - `/local/nx-displaygrid/nx-displaygrid.js?v=20260225-235900`

### Why it changed
- To match partner feedback from live review and remove remaining friction in ambient navigation, food workflows, and destructive settings actions.

   - `bash scripts/check_structure.sh`

   - Confirm bottom action row shows 3 equal buttons (`Heating`, `Lighting`, `Screensaver`).

   - In `Recipes`, add ingredients/steps via `+` controls and save recipe.
   - In `Meals`, select a meal day and click `Begin cooking`; confirm `Cooking` tab appears and step tick-offs persist.
   - Click `Finish cooking`; confirm `Cooking` tab disappears.
6. In Settings shopping favourites:
   - Click reset/clear and confirm two-step prompt appears with changed button order on step two.

## Family dashboard rebuild and UX fixes

### What changed
- Added a new Lovelace dashboard entry and YAML file for `NX - Family Dashboard`:
  - `config/configuration.yaml` (`lovelace.dashboards.nx-family-dashboard`)
  - `config/lovelace/nx-family-dashboard.yaml`
- Added family-mode navigation behavior in the card/frontend:
  - Sidebar base items for family mode are now `Calendar`, `Chores`, `Food`, `Family Dashboard`, `Ambient`.
  - Sidebar now includes a `+` action button that triggers the same add flow as the header action.
  - Admin menu screens remain admin-only and can be individually toggled from Settings (`family_dashboard_v3.admin_menu`).
- Reworked Ambient view for family mode:
  - Bin status in one line (`Bin day: ...`) with bin color icons and put-out-day messaging.
  - Home control collections (`Heating`, `Lighting`) open modal collections with on/off toggles.
  - Focus/landscape ambient mode hides topbar/sidebar and exits with double tap/double click.
  - Glance section now emphasizes important todo countdowns.
- Reworked Food view:
  - Added tabs for `Meals`, `Recipes`, `Shopping List`, and `In the house`.
  - Meal planner now supports search (not dropdown-only) via datalist input.
  - Added recipe create/edit with ingredients + instruction steps.
  - Added per-user 1-5 star rating/review storage.
  - Added add-single-ingredient and add-whole-recipe to shopping actions.
  - Added pre-add selection modal when adding meal/recipe ingredients to shopping.
- Updated shopping item merge parsing:
  - Name-based grouping now handles quantity/unit patterns and sums quantities with unit-aware normalization where available.
- Added settings support for configurable food units (`food_v2.units`).
- Updated add-event dialog:
  - Added all-day toggle.
  - Replaced problematic date/time interactions with explicit date/time picker rows and Safari-safe select styling.
- Month view now uses a scrollable container for “today events” under the calendar.
- Bumped Lovelace resource cache-bust query:
  - `config/lovelace/resources.yaml` -> `?v=20260225-214500`.

### Why it changed
- The existing v2 dashboard did not support the requested family-first IA and interaction model for ambient controls, food/recipes, and shopping workflows.

### How to verify
1. Run checks:
   - `bash scripts/check.sh`
2. Run structure guard:
   - `bash scripts/check_structure.sh`
   - If the host shell is Bash 3.x and `mapfile` is unavailable, run the equivalent manual structure check used in this session.
3. In Home Assistant, hard refresh Lovelace after deploy (resource query-string changed).
4. Confirm sidebar dashboard entry `NX - Family Dashboard` appears and loads.
5. Confirm family-mode sidebar items are present (`Calendar`, `Chores`, `Food`, `Family Dashboard`, `Ambient`) and `+` opens add dialog.
6. On Ambient:
   - Verify bin line reads as single-line status with icon(s),
   - Tap `Heating`/`Lighting` and confirm modal toggle controls work,
   - Tap focus button and verify topbar/sidebar hide; double tap/double click exits focus mode.
7. On Food:
   - In `Meals`, search and assign meals by name; add to shopping and verify deselect modal appears.
   - In `Recipes`, add/edit recipe ingredients and steps, rate as current user, add single ingredient and whole meal to shopping.
   - In `Shopping List`, verify shopping list remains usable inside Food view.
8. In add-event flow, test all-day creation and date/time picker behavior in Chrome and Safari.
9. In month view, confirm the today-events panel beneath the month grid scrolls when many events exist.
