# 2026-02-19

- Updated Home Controls layout in `config/www/nx-displaygrid/views/home.view.js` to use `repeat(auto-fit, minmax(260px, 1fr))` with 12px gaps.
- Added shrink-safe grid/tile guards (`.grid > * { min-width: 0; }`, tile `box-sizing: border-box`) to prevent horizontal card collisions.
- Added strict tile containment (`position: relative`, `overflow: hidden`) and a shrink-safe content wrapper to stop long-label bleed that looked like tile overlap.
- Removed the legacy Settings "People wizard" UI entry and modal so only onboarding controls people/source setup flow.
- Bumped Lovelace resource URL version in `config/lovelace/resources.yaml` to force browser reload of updated JS.
- Added a Settings button to open the onboarding wizard directly (no reset required), while clearing the forced wizard mode automatically on finish.

Why:
- Home Control tiles could visually overlap at common desktop widths due to rigid column sizing under constrained space.
- Long/overflowing tile content could bleed into neighbouring cards, creating visual stacking/overlap even when the grid itself was valid.
- A second legacy wizard in Settings was still visible and conflicted with requested onboarding-only behavior.
- After removing the legacy wizard, there was no direct way to launch onboarding from Settings without a destructive reset.

How to verify:
1. Open the Home Controls view with at least 4 configured controls.
2. Resize the browser from wide desktop to tablet and phone widths.
3. Confirm controls reflow without overlap: up to 4 columns on wide screens, fewer columns as width shrinks, and 1 column on narrow widths.
4. Confirm long entity names wrap inside each tile and do not force horizontal overflow.
5. Confirm no tile content visually draws over adjacent tiles at ~900–1200px widths.
6. Open Settings and confirm the old "People wizard" add/edit flow is no longer shown.
7. In Settings -> People, click "Open setup wizard" and confirm onboarding opens immediately.
8. Complete wizard and confirm it exits back to normal views after finishing.
9. Reload dashboard and confirm the new resource `?v=20260219-191500` is requested.
