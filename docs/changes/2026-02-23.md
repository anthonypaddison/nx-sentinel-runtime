# 2026-02-23

## Commit 1 - Deployment/Docs Hardening

- Added `deploy/backup-remote-config.sh` for full remote `/config` snapshots (including `.storage`, excluding DB/logs, excluding secrets by default).
- Added `deploy/export-nx-displaygrid-setup.sh` to export the runtime `nx-displaygrid` setup (`.storage/nx_displaygrid.config` + relevant dashboard/integration files).
- Added optional pre-deploy backup behavior (enabled by default, configurable with `BACKUP_BEFORE_DEPLOY`, `HA_BACKUP_ROOT`, `HA_BACKUP_INCLUDE_SECRETS`).
- Added `scripts/check.sh` to run the repo's fast test/check workflow and skip unavailable tools gracefully.
- Removed stale legacy CI/tooling configs (`.travis.yml`, PHP/SQL lint configs) and simplified `.eslintrc.json` to match the actual JS-only repo.
- Fixed documentation drift across `README.md`, `docs/CONTRIBUTING.md`, `docs/RELEASE_CHECKLIST.md`, `ARCHITECTURE.md`, and `AGENTS.md`.
- Removed the stale resource-version comment from `config/lovelace/nx-displaygrid.yaml`.
- Updated `config/secrets.yaml.template` to include currently used secret keys.

Why:
- Deployment and data-structure changes needed a repeatable backup/export workflow so runtime setup can be restored without manual re-onboarding.
- Stale CI/tooling/docs were increasing onboarding time and reducing trust in the runbook.

How to verify:
1. Run `scripts/check.sh` and confirm Node tests pass (tool-dependent checks may be skipped if not installed).
2. Run `bash -n deploy/*.sh scripts/check.sh` and confirm no shell syntax errors.
3. Run `deploy/backup-remote-config.sh <host> lab` against a reachable HA host and confirm a timestamped snapshot is created under `backups/ha/`.
4. Run `deploy/export-nx-displaygrid-setup.sh <host> lab` and confirm the export includes `config/.storage/nx_displaygrid.config` when present.
5. Confirm `docs/RELEASE_CHECKLIST.md` references `config/lovelace/resources.yaml` for cache-busting.
