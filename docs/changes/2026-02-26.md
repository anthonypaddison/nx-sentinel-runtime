# 2026-02-26

## Food favourites + chip layout polish

### What changed
- Food view navigation now includes a dedicated `Favourites` tab.
- Added favourites panel toggle at the top of the panel:
  - `Favourite meals` (recipes with average rating `>= 3`).
  - `Favourite shopping` (items starred in shopping).
- Improved Food weekly meal-plan row responsiveness:
  - Per-day actions (`Add to shopping`, `Begin cooking`) are grouped in a responsive action block.
  - Layout now reflows cleanly on narrower widths to avoid squashed controls.
- Split shopping concerns in Food:
  - `Shopping List` remains separate.
  - Saved shopping-list bundles are rendered in shopping context instead of crowding the meal-plan panel.
- Updated topbar person chips (used in schedule/important/chores contexts):
  - Fixed 4 equal-width chips per row.
  - Counts moved below the name in a stacked layout.
  - Long names now clip with ellipsis to avoid chip distortion.
  - On mobile widths, chips switch to circular avatar mode with bold first-letter initials while preserving active-toggle highlight behavior.
- Updated Lovelace resource cache-bust:
  - `/local/nx-displaygrid/nx-displaygrid.js?v=20260226-134700`

### Why it changed
- To stabilise layout on smaller screens and implement the requested favourites split/toggle behavior while keeping chip rows visually consistent with long names.

### How to verify
1. Run checks:
   - `bash scripts/check.sh`
   - `bash scripts/check_structure.sh`
2. Hard refresh the browser after deploy (cache-bust updated).
3. In Food view:
   - Confirm tabs include `Meals`, `Recipes`, `Shopping List`, `Favourites` (and `Cooking` only when active).
   - In `Favourites`, switch between `Favourite meals` and `Favourite shopping` using the top toggle.
   - Confirm favourite meals list only includes recipes with average rating >= 3.
   - Confirm favourite shopping list shows starred shopping entries and can add/unstar.
4. In Meals view, reduce viewport width and confirm day rows/actions no longer collapse into unusable narrow buttons.
5. In schedule/chores/important top chips:
   - Confirm 4 chips per row with equal width.
   - Confirm counts are displayed beneath each name.
   - Confirm long names are ellipsized and no chip overlap/squash occurs.
   - On a narrow/mobile viewport, confirm chips become circles with initial letters and active toggles remain visually highlighted.
