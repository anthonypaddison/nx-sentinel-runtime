# 2026-02-24

## Commit 1 - Websocket Config Write Hardening

- Added `@websocket_api.require_admin` to `nx_displaygrid/config/set` so only HA admins can save shared `nx-displaygrid` config.
- Added backend payload sanitization/limits for websocket config writes (type checks, nesting depth, key/item counts, string length, payload size).
- Bumped the in-repo custom integration manifest version to `0.1.1` after backend behavior changes.

Why:
- Non-admin users should not be able to overwrite shared dashboard config; write access needed HA permission enforcement at the custom integration boundary.
- Payload validation reduces storage corruption risk from malformed or unexpectedly large websocket writes.

How to verify:
1. Run `python3 -m py_compile config/custom_components/nx_displaygrid/__init__.py config/custom_components/nx_displaygrid/websocket.py`.
2. Run `bash scripts/check.sh` and confirm the Node test suite still passes.
3. In Home Assistant, try saving `nx-displaygrid` config as a non-admin user and confirm the websocket write is rejected.
4. As an admin user, save a normal settings change and confirm it still persists.

Version assumptions:
- Reviewed Home Assistant developer websocket decorator docs and Home Assistant 2026.2 release notes on 2026-02-24.
- Assumes `@websocket_api.require_admin` remains the supported permission guard for custom websocket commands in the current HA Core series.
