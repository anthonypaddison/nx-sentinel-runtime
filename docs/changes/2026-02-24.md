## Commit 2 - Admin PIN Hashing and Export Redaction

- Added `config/www/nx-displaygrid/util/admin-pin.util.js` to hash and verify admin PINs with a salted SHA-256 format (`sha256$<salt>$<hash>`).
- Updated admin unlock/set flows to verify hashed PINs, migrate legacy plaintext PINs after a successful unlock, and store new PINs as hashes.
- Updated the Settings unlock button handler to await async PIN verification.
- Stopped including `admin_pin` in YAML export serialization by default and added tests covering PIN hashing + export redaction.

Why:
- Plaintext admin PINs were being persisted in shared config/local storage and could leak through YAML export/copy flows.
- A compatibility-preserving migration avoids breaking existing setups while improving stored secret hygiene.

How to verify:
1. Run `node --test tests/*.test.mjs` and confirm the new admin PIN tests pass.
2. In Home Assistant, set or change the admin PIN in `nx-displaygrid` Settings and confirm admin unlock still works on this device.
3. If you have an older plaintext PIN stored, unlock once and confirm future unlocks still work (legacy value should be rewritten as a hash).
4. Use any nx-displaygrid YAML export/copy action and confirm `admin_pin:` is not included in the exported YAML.

Version assumptions:
- Reviewed Home Assistant frontend/custom-card docs and Home Assistant 2026.2 release notes on 2026-02-24; no HA API changes assumed for this client-side PIN storage hardening.
